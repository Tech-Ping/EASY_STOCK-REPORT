# Project-Design & Implementation
| 항목 | 내용 |
|:---  |---  |
| (1) 요구사항 정의 | 🤖 **<u>AI 챗봇 ‘스토기’</u>: 주식 용어 및 개념에 대한 질의응답 지원** <br>- GPT-4 Turbo 기반<br>- 프롬프트 튜닝을 통해 금융/주식 용어를 초보자가 이해하기 쉬운 문장으로 설명<br>- 벡터 DB에 적재된 데이터를 기준으로 주식 도메인 맞춤형 답변 제공 및 질의응답부터 실시간 주가 정보 연동까지 대응<br><br>📈 **<u>모의투자 기능</u>: 실제 시장 데이터 기반의 가상 투자**<br>- 실시간 주식 데이터 제공 (`네이버 증권 API`, `한국투자증권 Open API` 등 활용)<br>- 자체 가상화폐 'STOCKEN'으로 매수/매도 기능 구현<br>- 사용자별 포트폴리오 저장 및 수익률 계산 기능 포함<br><br>**🧠 <u>튜토리얼 & 퀘스트 학습 기능</u>: 퀘스트 기반의 주식 학습 시스템** <br>- 난이도별 커리큘럼 기반 퀘스트 문제 제공<br>- 퀘스트 수행 시 보상(STOCKEN) 지급<br>- 퀘스트 이력은 DB 테이블로 관리<br><br>**📊 <u>투자 리포트 기능</u>: 월간 투자 요약 리포트 자동 제공**<br>- 매월 1일 Batch Job으로 지난 달의 리포트 생성<br>- DB 테이블에 JSON 데이터로 실 수익률-총 투자금 데이터 저장<br>- 수익률/수익금 계산 및 투자 성향 분석 포함<br><br>📰 **<u>관심 종목 뉴스 요약 및 푸시 알림</u>: 사용자가 북마크한 종목에 대한 최신 뉴스 요약 및 알림 제공**<br>- Naver 금융 뉴스 크롤링 (Selenium + BeautifulSoup)<br>- LangChain을 이용한 뉴스 요약<br>- Pinecone 벡터 DB 기반 검색 최적화<br>- Firebase Cloud Messaging (FCM)을 통한 뉴스 알림  전송  <br></br> **[ER Diagram]** ![EASY-STOCK](https://github.com/user-attachments/assets/04eaa36f-c63a-4543-b12d-75b267ef6658)|
| (3) 주요엔진 및 기능 설계 |본 프로젝트는 AI 기반 질의응답, 투자 시뮬레이션, 리포트 생성, 뉴스 요약 등 복합 기능을 포함하며, 각 기능은 모듈화되어 작동합니다.<br><br> ![image](https://github.com/user-attachments/assets/92a9cb61-a09b-4a48-b0bd-9bc4cd50351e)<br> **1. 사용자 인터페이스 (Client Layer)**<br>  - 사용자 질의 입력 또는 매수/매도 요청<br><br>  **2. AI 챗봇 모듈 (LLM + Vector DB)**<br>  - 사용자의 질문을 Vector DB에서 문서 검색(Retrieval)<br>  - 가장 연관도 높은 문서를 기반으로 GPT-4 Turbo에게 프롬프트 전송<br>  - 주식 개념 설명, 투자 리포트 해석 등 도메인 특화 응답 제공<br><br>  **3. 투자 리포트 생성 모듈 (Batch + RDBMS)**<br>  - 매월 1일 배치 작업으로 사용자별 투자 리포트 생성<br>  - 수익률, 수익금, 투자 성향 분석 결과를 JSON 형태로 DB에 저장<br>  - Vector DB에 투자 리포트 내용을 적재하여 검색/요약에 활용<br><br>  **4. 투자 비즈니스 로직 모듈 (Spring Boot 기반 Core API)**<br>  - 실시간 주식 시세 기반 매수/매도 요청 처리<br>  - STOCKEN 기반 가상 투자 수행 및 거래 내역 RDB에 저장<br>  - 종목 북마크, 퀘스트 보상, 포트폴리오 관리 기능 포함<br><br>  **5. 뉴스 요약 모듈 (크롤러 + LangChain + Pinecone)**<br>  - 네이버 뉴스 API 기반 주요 Ticker 뉴스 수집 (Batch Job)<br>  - 수집 뉴스는 LangChain을 통해 요약<br>  - 요약 뉴스는 Pinecone 벡터 DB에 저장되어 RAG에 활용<br>  - 북마크 종목에 대한 푸시 알림 전송 (FCM)<br><br>  **6. 데이터베이스 구성**<br>  - RDBMS (MySQL): 사용자 정보, 투자 내역, 퀘스트 등 저장<br>  - Pinecone: 요약 뉴스, 투자 리포트 등 벡터 기반 문서 검색용 저장소<br><br> <br> ![파이프라이닝 아키텍처](https://github.com/user-attachments/assets/73801da4-1c73-40d5-8ce7-4372324147f3) |
| (4) 주요 기능의 구현 |  <strong>1. 🤖 AI 챗봇 ‘스토기’ – GPT-4 Turbo 기반 주식 질의응답 처리</strong><br>  - 사용자는 `/ask` API를 통해 주식 용어나 투자 관련 질문을 입력함<br>  - Flask 서버는 해당 질문을 Pinecone 기반 벡터 DB에 질의하여 관련 문서를 검색<br>  - 검색된 문서 내용을 기반으로 GPT-4 Turbo에게 프롬프트를 구성하고 응답 생성<br>  - 예: “PER이 뭐야?”라는 질문에 대해 스토기는 `“주가를 주당순이익(EPS)으로 나눈 지표로, 낮을수록 저평가된 주식일 가능성이 있습니다.”`와 같은 설명을 반환<br>  - 결과는 웹 UI를 통해 사용자에게 출력됨<br><br>  <strong>2. 모의투자 기능 – STOCKEN 기반 매수/매도 처리</strong><br>  - 사용자는 `/trade/buy` 또는 `/trade/sell` API를 통해 특정 종목에 대해 가상 투자 요청을 보냄<br>  - 현재 주가는 `한국투자증권 Open API`를 통해 조회됨<br>  - 서버는 사용자의 보유 STOCKEN과 실시간 주가를 검증한 뒤 주문을 처리함<br>  - 거래가 완료되면 거래 내역은 stock_record 테이블에 저장되며, 보유 종목 정보는 `inventory` 테이블에 반영됨<br>  - 이 데이터는 이후 월간 투자 리포트 분석 및 수익률 계산에 활용됨 |
